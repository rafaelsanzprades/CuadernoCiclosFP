## Arquitectura de datos

### Tres capas de datos

1. **Catálogo** (BOE/BOA): Ciclos formativos, módulos, RA, CE, UD
2. **Datos DEMO**: Scripts de seed con curso completo, alumnos ficticios, seguimiento
3. **Datos reales**: Archivos locales `.cddp` (Programación) y `.cddc` (Curso)

### Principio clave

El servidor web **NUNCA** almacena datos reales del docente. Solo gestiona catálogos oficiales y datos DEMO para pruebas.

## Stack tecnológico

- **Frontend**: Next.js 16.2.7, React 19, TypeScript, Tailwind CSS v4, Turbopack
- **Backend**: Python 3.13, FastAPI, SQLite, ADOdb
- **Tests**: Playwright E2E (18 tests × 3 navegadores = 54 tests)
- **PDFs**: Python scripts para generación de PDFs y Excel
- **Datos**: Archivos locales `.cddp` / `.cddc`, catálogos en `src/data/curriculos/`
